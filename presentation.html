Alice, Bob, and Carol

Alice works for a small company, and her boss just asked her to get a bunch of data from AliceBane.com. She's becoming overwhelmed at the amount of data and how many times she's going to have to click to get all the data they need.

Bob works for a slightly larger company that makes a web application that keeps breaking. He has been assigned to figure out how to test the site and keep it reliable, and is starting to feel bored going through the same steps over and over to test each iteration of the application.

Carol is friends with both Alice and Bob, and is about to help both of them out, but she doesn't know that yet.

The Meeting

Alice, Bob, and Carol go to lunch together one day, and Alice starts talking about her problem first. Bob pipes up, and they go back and forth for a bit about how repetitive their tasks are and how long they'll take, until Carol manages to find a break in the conversation and says, "Have you considered Selenium?"

"What's Selenium?" they both replied in unison.

"Selenium is a tool for automating manual browser tasks. You can use it from most popular programming languages, and it provides an API that you can use to automate your repetitive tasks, getting the information you need, Alice, and checking that the parts of your application are working properly, Bob."

A Simple Example (2)

>>> from selenium import webdriver
>>> keys = webdriver.common.keys.Keys()
>>> driver = webdriver.Firefox()
>>> driver.get('http://www.google.com/')
>>> search_field = driver.find_element_by_name('q')
>>> search_field.send_keys('koala')
>>> search_field.send_keys(keys.ENTER)
>>> driver.find_element_by_link_text('Images').click()

How to choose

Interactions on elements

  • Check whether it is displayed, enabled, or selected
  • Clear
  • Click
  • Find the parent element or elements within it
  • Get attribute, CSS property, ID, size, tag name, text
  • Get location with or without scrolling into view
  • Send keys
  • Submit

More complex interaction, such as click and drag, can be performed using action chains.

About timing

Not all pages take the same amount of time to load, and interactions within a page can also take different amounts of time.

When finding an element, Selenium honours an implicit wait value that is set globally for the webdriver instance. By default, it is set to three seconds, and you can set it differently if you expect that most of your element searches will take a different amount of time.

For finer control, you can use explicit waits, allowing you to specify timing for particular lookups without changing the global implicit wait time.
